实操ftp遇到的问题

一、文件无法上传

  • 辨别服务器端ftp主动与被动模式
    检查/etc/vsftpd/vsftpd.conf
    connect_from_port_20=YES #主动模式
  • 主动模式下，确保防火墙打开了端口20
    firewall-cmd --permanent --add-port=20/tcp
    firewall-cmd --reload #激活修改的配置
    也可以选择关闭防火墙
    systemctl stop firewalld
  • 转移目录所有权,将目标目录所有权给root用户
    #假设向/home目录传输文件
    ll /home
    chown -R root /home
  • 确保/etc/vsftpd/vsftpd.conf配置中
    write_enable=YES

二、无法登录

  • 更改配置vsftpd.conf
    #/etc/vsftpd/vsftpd.conf
    userlist_enable=NO
  • 检查ftpusers,删除要登录的用户名
  • 检查user_list,添加要登录的用户名
  • 重启
    systemctl restart vsftpd

三、Mac终端找不到ftp命令

  • 安装ftp等工具
    brew install inetutils

四、一些概念

  • 主动FTP:
    命令连接：客户端 大于1024端口 >> 服务器 21端口
    数据连接：服务器 客户端（大于1024的端口）<< 20端口
  • 被动FTP：
    命令连接：客户端大于1024端口 >> 服务器 21端口
    数据连接：客户端大于1024端口 >> 服务器大于1024端口

linux►防火墙

十、防火墙
启动 systemctl [] firewalld
查看 firewall-cmd –list-all
加服务 firewall-cmd –permanent –add-service=http
加端口 firewall-cmd –permanent –add-port=8080/tcp
–remove-port=8080/tcp
重载 firewall-cmd –reload
@see ftp
若启动了防火墙,客户机要ftp被动连接服务器告知的端口,那么服务器不仅要开放21端口，还要开放1025–5000的所有端口;此时用ftp主动链接即可
